## Key Decisions
- **Events pattern, not Checker ABC**: `checkers/flights.py` has no `register()`, not discovered by `checkers.discover()` — intentional, follows smarticket/kehilatayim model
- **All airlines by default** (`airlines: {}`): empty dict = watch all. User can exclude specific IATA codes via `/flights` panel. Previously defaulted to LY/IZ/6H only — changed because the hotel/charter market is broader.
- **01:00–06:00 default window**: user explicitly requested early-morning departure window (cheapest flights, family travel)
- **Kayak over airline sites**: airline deep links are fragile and airline-specific. Kayak has stable URL format and shows all options. `kayak.com` used (not `.co.il`) for consistency.
- **Round-trip Kayak link** (trip_days=4 default): the linter extended `booking_url` to include return date; 4 days is a reasonable short-trip default. Not configurable in UI yet.
- **No seat availability check**: FR24 has no availability data. Kayak deep link pre-fills 2 adults + 1 lap infant so user can verify in 1 click.
- **2h lead time**: flights departing < 2h from now are silently skipped — no booking is realistically possible.

## Open Questions / Blockers
- **`trip_days=4` not configurable**: the return date on the Kayak link is hardcoded to departure + 4 days. Consider adding to filter settings if user travels for different durations.
- **FR24 API is unofficial**: no auth, no SLA. The exponential backoff handles transient failures, but the API could change shape or add bot detection. Monitor logs for "FR24: fetch failed" errors.
- **First-run burst**: on fresh deploy (empty `known_flight_ids`), the first poll alerts on ALL matching flights. Acceptable one-time behavior, but worth noting if re-deploying.
- **`state.json` hour defaults**: if the bot ran before session 2, `filter_settings` may already be persisted with old values (0/23). User should open `/flights` and tap the `01-06` preset to override. The code defaults only apply when `filter_settings` is absent from state.
